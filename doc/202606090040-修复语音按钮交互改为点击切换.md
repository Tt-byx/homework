# 修复语音按钮交互：按住说话改为点击切换

## 提示词

> http://localhost:5173/chat 按住说话的交互功能有问题，会调用系统麦克风，但是没有后续操作如发送暂停等

## 做了什么

### 1. 问题诊断

原代码用 `@mousedown`/`@mouseup` 实现"按住说话"模式，但：
- Element Plus 的 `el-button` 组件内部会吞掉原生鼠标事件
- `@mouseleave` 导致鼠标稍有移动就取消录音
- 用户体验差：无法看到明确的录音开始/停止状态

### 2. 修复方案

改为**点击切换模式**：
- 点击"语音输入" → 开始录音（按钮变红，显示波浪动画 + 计时）
- 再点"停止录音" → 停止录音，自动发送

关键改动：
- 去掉 `@mousedown`/`@mouseup`/`@mouseleave`，改用 `@click="toggleRecording"`
- 新增 `toggleRecording()` 函数：根据 `isRecording` 状态切换开始/停止
- 录音状态指示器增加"点击停止录音发送"提示

### 3. 验证

- ✅ `npm run build` — built in 883ms

## 为什么这样做

"按住说话"在 Web 端不可靠（鼠标移动会触发 mouseleave）。点击切换是更稳定的交互模式。

## 产生的结果

语音按钮交互改为：点一下开始录音 → 再点一下停止并发送。状态指示清晰。
