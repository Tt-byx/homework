# 修复：知识库大文件处理失败 + 两步式进度条验证

## 提示词

> 上传文件后点处理文件后会失败，你可以试一试

## 做了什么

### 问题诊断

xlsx 文件包含 5.17 亿字符，产生 170 万个 chunks，调 SiliconFlow embedding API 触发 429 限流。

### 修复

1. **text_chunker.py** — 新增 `max_chunks=2000` 参数，超长文本自动截断
2. **embedding_service.py** — 添加 3 次重试 + 指数退避（2s/4s/8s）应对限流

### 端到端测试结果

```
上传 xlsx → vectorStatus=0（待处理）
点击处理 → parsing 5% → chunking 30% → embedding 35%→85% → storing 100%
完成 → vectorStatus=2, chunks=2618
```

## 编译验证

- ✅ Java `mvn compile` — BUILD SUCCESS
- ✅ 前端 `npm run build` — built in 1.35s
