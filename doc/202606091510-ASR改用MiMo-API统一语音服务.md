# ASR 改用 MiMo API，统一语音服务

## 提示词

> 所以我tts和asr语音识别都是用的小米的？阿里云的那个语音识别没用了？你觉得用哪一个好，列出优缺点/然后这个音色我能不能自定义？

## 做了什么

### 1. ASR 从 FunASR 改为 MiMo ASR API

**改动文件：** `asr_service.py` — 完全重写

原方案：FunASR 本地模型（paraformer-zh），需要 funasr + torch 依赖，首次加载 35 秒
新方案：MiMo `mimo-v2.5-asr` API，零依赖，即时响应

MiMo ASR 的消息格式特殊：需要用 `input_audio` 类型（非普通字符串），通过 httpx 手动发送。

### 2. 当前统一架构

| 能力 | 服务 | 来源 |
|------|------|------|
| ASR 语音识别 | `mimo-v2.5-asr` | MiMo API |
| TTS 语音合成 | `mimo-v2.5-tts` | MiMo API |
| LLM 大模型 | `mimo-v2.5-pro` | MiMo API |
| Embedding | `BAAI/bge-m3` | SiliconFlow API |

全部用 API 调用，无需本地部署任何模型（ChromaDB 除外）。

### 3. 附带改动

- `main.py` — 去掉 ASR 模型预加载（不再需要）
- `chat.py` — 去掉 webm→wav 转换函数（MiMo ASR 可直接处理）

### 4. 验证

- ✅ MiMo ASR API 调用成功
- ✅ MiMo TTS API 调用成功（53KB WAV）
- ✅ `npm run build` — built in 928ms

## 为什么这样做

FunASR 本地模型依赖重（funasr + torch 约 3-5GB）、首次加载慢（35 秒）、受代理阻断影响。MiMo API 零依赖、即时响应、与 TTS/LLM 统一管理。

## 产生的结果

项目语音服务全部统一为 MiMo API，架构极简，无需本地模型部署。
