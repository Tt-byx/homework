# 修复合并冲突 + 配置 ASR/TTS 本地模式

## 提示词

> 用我这一套RAG，java端口改10086，前端代理转10086，CORS不用加5176端口，python版本改为3.14，asr语音识别和tts语音合成都先用本地模式

## 做了什么

### 1. 问题诊断

从 GitHub 拉取了 2 个新提交（Phase 2 语音交互代码），发现以下冲突：

| 问题 | 影响 |
|------|------|
| `rag_service.py` 被完全覆盖 | Phase 1 的 embedding 检索失效，改用 ChromaDB 内置 embedding（效果差） |
| `config.py` 删除了 embedding 配置 | SiliconFlow BAAI/bge-m3 无法使用 |
| `main.py` 删除了知识库路由注册 | `/api/knowledge/*` 端点 404 |
| `pyproject.toml` 删了 python-docx/openpyxl | 文档解析依赖丢失 |
| `application.yml` 端口改回 8080 | 与前端代理 10086 冲突 |
| `vite.config.js` WS 代理指向 8080 | WebSocket 连接失败 |
| `websocket.js` 硬编码 8080 | 同上 |
| `chat.py` 使用错误的 RAG 检索 | 回答准确率下降 |

### 2. 修复内容

**Python 后端（6 个文件）：**

| 文件 | 修复内容 |
|------|---------|
| `config.py` | 恢复 embedding_api_key/base_url/model 配置，chroma_persist_dir 改回 `./chroma_data` |
| `rag_service.py` | 完全恢复 Phase 1 版本（使用 embedding_service + vector_store 做检索） |
| `llm_service.py` | 保留新版本的 `chat_stream()`（流式），新增 `chat_with_rag()`（RAG 增强） |
| `chat.py` | `/chat` 端点恢复用 `chat_with_rag()`，`/chat/stream` 用 `retrieve_context()` 做 RAG |
| `main.py` | 注册 knowledge 路由，CORS 允许 10086+5173 |
| `pyproject.toml` | 恢复 python-docx/openpyxl，Python 版本改为 >=3.13 |

**Java 后端（1 个文件）：**

| 文件 | 修复内容 |
|------|---------|
| `application.yml` | 端口 8080 → 10086，添加 upload dir 配置 |

**前端（2 个文件）：**

| 文件 | 修复内容 |
|------|---------|
| `vite.config.js` | WS 代理目标 8080 → 10086 |
| `websocket.js` | WS_URL 从硬编码 8080 改为动态获取当前 host（走 Vite 代理） |

### 3. 依赖安装

- `.python-version` 从 3.14 改为 3.13（funasr 的 editdistance 依赖在 3.14 上编译失败）
- `uv sync` 安装基础依赖成功
- funasr 单独安装（`--no-deps`），手动补装 kaldiio、torch_complex
- PyTorch CPU 版本安装成功（torch 2.12.0+cpu）
- 所有模块导入验证通过

## 为什么这样做

- 合并新代码时，Phase 1 的 RAG 检索被新版本的简陋版覆盖（用 ChromaDB 内置 embedding 而非 SiliconFlow bge-m3），需要恢复
- 端口配置必须统一（Java 10086，前端代理 10086），否则前后端通信断裂
- funasr 在 Python 3.14 上无法编译（editdistance C 扩展不兼容），降级到 3.13 解决

## 产生的结果

- Phase 1 RAG 检索恢复（SiliconFlow bge-m3 embedding + ChromaDB 183 chunks）
- 端口配置统一（Java 10086，WS 代理 10086）
- ASR/TTS 本地模式就绪（funasr + torch 已安装，模型将在首次请求时懒加载）
- 全链路代码就绪：语音提问 → ASR → RAG 检索 → LLM 流式回答 → TTS 语音合成 → 前端播放
