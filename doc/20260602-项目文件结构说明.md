## 一、项目整体架构

本项目采用**前后端分离**架构，包含三个核心模块：

```
┌─────────────────┐     HTTP/WS      ┌─────────────────┐     HTTP      ┌─────────────────┐
│   前端 (Vue3)    │ ◄──────────────► │  Java后端        │ ◄───────────► │  Python后端      │
│   端口: 5173     │                  │  (Spring Boot)   │               │  (FastAPI)       │
│                  │                  │  端口: 8080       │               │  端口: 8000       │
└─────────────────┘                  └────────┬────────┘               └────────┬────────┘
                                              │                                 │
                                              ▼                                 ▼
                                         ┌─────────┐                    ┌──────────────┐
                                         │  MySQL   │                    │  MiMo API    │
                                         │  数据库   │                    │  (大模型)     │
                                         └─────────┘                    └──────────────┘
```

**核心原则：**
- **前端**：只与 Java 后端通信，不直接访问 Python 后端
- **Java 后端**：面向前端的唯一入口，管理所有业务数据
- **Python 后端**：只负责 AI 能力（大模型调用），被 Java 后端调用

## 三、各模块文件详细说明

### 3.1 前端模块 (frontend/)

前端使用 **Vue 3 + Element Plus + Vite** 构建，负责用户界面展示和交互。

#### 3.1.1 配置文件

| 文件路径 | 用途说明 |
|---------|---------|
| `frontend/package.json` | 项目依赖配置，定义了 Vue3、Element Plus、Pinia、Axios 等依赖包 |
| `frontend/vite.config.js` | Vite 构建工具配置，设置了路径别名 `@`、开发服务器端口 5173、API 代理（将 `/api` 请求代理到 Java 后端 8080 端口） |
| `frontend/index.html` | 入口 HTML 文件，Vue 应用的挂载点 |
| `frontend/.gitignore` | Git 忽略规则，排除 node_modules、dist 等目录 |

#### 3.1.2 入口文件

| 文件路径 | 用途说明 |
|---------|---------|
| `frontend/src/main.js` | Vue 应用入口，初始化应用并注册全局插件（Element Plus、Pinia 状态管理、Vue Router 路由） |
| `frontend/src/App.vue` | 根组件，仅包含 `<router-view />` 用于渲染当前路由对应的页面 |

#### 3.1.3 路由配置

| 文件路径 | 用途说明 |
|---------|---------|
| `frontend/src/router/index.js` | 路由配置文件，定义了两个主要路由组：<br>1. **游客端** `/`：首页（HomeView）和聊天页（ChatView）<br>2. **管理后台** `/admin`：数据大屏（DashboardView）和知识库管理（KnowledgeView） |

#### 3.1.4 API 请求层

| 文件路径 | 用途说明 |

### 3.2 Java 后端模块 (backend-java/)

Java 后端使用 **Spring Boot 3.3.5 + MyBatis-Plus + MySQL** 构建，是面向前端的唯一网关。

#### 3.2.1 配置文件

| 文件路径 | 用途说明 |
|---------|---------|
| `backend-java/pom.xml` | Maven 项目配置，定义了项目依赖：<br>- Spring Boot Web（Web 服务）<br>- Spring Boot WebSocket（WebSocket 支持）<br>- MyBatis-Plus（数据库 ORM）<br>- MySQL Connector（数据库驱动）<br>- Lombok（简化代码） |
| `backend-java/src/main/resources/application.yml` | 应用配置文件：<br>- 服务端口：8080<br>- MySQL 连接配置（数据库：scenic_ai）<br>- MyBatis-Plus 配置（驼峰命名转换、日志输出）<br>- Python 后端地址：http://localhost:8000 |

#### 3.2.2 启动类

| 文件路径 | 用途说明 |
|---------|---------|

## 六、开发注意事项

1. **启动顺序**
   - 先启动 MySQL 数据库
   - 启动 Python 后端：`cd backend-python && uvicorn main:app --reload`
   - 启动 Java 后端：`cd backend-java && mvn spring-boot:run`
   - 启动前端：`cd frontend && npm run dev`

2. **环境变量**
   - Python 后端需要配置 `.env` 文件，包含 `MIMO_API_KEY`

3. **端口占用**
   - 前端：5173
   - Java 后端：8080
   - Python 后端：8000
   - MySQL：3306
