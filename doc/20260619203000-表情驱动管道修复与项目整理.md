# 20260619-表情驱动管道修复与项目整理

## 提示词

用户原始需求分为两部分：
1. "情感分析分为两个：(1) AI数字人感知游客情绪并据此回答 (2) AI数字人根据自己的回复做出表情如笑、哭、惊讶"
2. "整理一下我的项目然后传到GitHub"

## 做了什么

### 一、表情驱动管道打通

**问题**：Python 后端发送 `expression` SSE 事件 → Java WebSocket 转发 → 前端 `chatStore.currentExpression` 已更新，但 Live2D 模型从未收到 `setExpression()` 调用——表情管道在最后一步断了。

**修复**：
- ChatView 新增 watcher 监听 `chatStore.currentExpression`，调用 `live2dRef.value.setExpression()`
- VoiceCallView 同样新增表情驱动 watcher
- 新增 `applyExpression()` 直接回调，WebSocket 收到 expression 事件时立即调用

### 二、表情重置时机修正

**问题**：问完"迷路了怎么办"（Cry）后再问"推荐路线"，表情仍是 Cry。

**根因**：原方案在 `loading→false` 时延迟 2 秒重置，但新请求的 `loading→true` 比旧的延迟重置先触发。

**修复**：
- `loading→true`（新请求开始）：立即重置为 Normal
- `loading→false`（请求结束）：2 秒安全兜底

### 三、Live2D 模型层表情残留修复（核心难点）

**问题**：全链路日志确认 `setExpression('Smile')` 被正确调用，但视觉仍是 Cry。

**根因分两层**：

**层1 - 身体参数残留**：`_applyExpressionBodyMotion` 用 `addParameterValueById`（累加）设置身体参数，切换表情时不重置。Cry 设置了 `BodyAngleX=0.5, ShoulderX=-0.3`，切到 Smile 时只覆盖了 `BodyAngleY` 和 `BodyAngleX2`，其余保留 Cry 的值。

**修复**：切换表情前先用 `setParameterValueById` 把 5 个身体参数全部重置为 0。

**层2 - 表情动画残留（关键）**：`_expressionManager.startMotion(smileMotion)` 不会停止 Cry 的 `.exp3.json` 动画——Cry 动画继续淡出，期间 Cry 的面部参数（眼睛、嘴巴）仍在每帧叠加，覆盖了 Smile 效果。

**修复**：
- `CubismExpressionMotionManager` 新增 `resetAll()` 方法：清空旧动画队列、淡出权重缓存、表情参数缓存
- `LAppModel.setExpressionByName()` 启动新动画前先调用 `resetAll()`

### 四、项目整理与推送

- 清理所有调试用 `console.log` 和 `window.__testExpression`
- 保留原有有用的日志（WebSocket 连接、音频播放）
- 编译验证前端 + Java 后端
- 全部 7 个 commit 推送到 GitHub

## 完整的情感管道架构

```
游客输入
    ↓
Python analyze_sentiment(input) → 感知游客情绪(positive/negative/neutral)
    ↓ 注入 LLM 系统提示词
    ↓ "根据游客情绪调整回答风格（如游客沮丧时给予鼓励）"
LLM 生成回复
    ↓
Python detect_expression(reply) → 关键词检测 AI 回复情感
    ↓ Smile/Cry/Angry/Star/Normal
SSE event: expression → Java WebSocket 转发
    ↓
前端 chatStore._expressionCallback(expr)
    ↓ 直接调用（跳过 watcher 时序问题）
live2dRef.setExpression(expr)
    ↓
useLive2D.setExpression → ExpressionMap 映射
    ↓
LAppModel.setExpressionByName
    ├── _expressionManager.resetAll()        ← 清空旧动画
    ├── _expressionManager.startMotion(new)   ← 启动新表情动画
    └── _applyExpressionBodyMotion
        ├── _resetBodyParams()                ← 重置身体参数
        ├── _hideAllParts()                   ← 隐藏特效部件
        └── switch: 设置新表情的参数和部件
```

## 结果

表情系统现在完全可用：
- AI 回复开心内容 → 数字人微笑（Smile）
- AI 回复抱歉/遗憾内容 → 数字人难过（Cry）
- AI 回复警告内容 → 数字人生气（Angry）
- AI 回复好奇内容 → 数字人惊讶（Star）
- 切换不同表情时干净利落，无残留
- 每轮对话结束后自动恢复默认微笑
