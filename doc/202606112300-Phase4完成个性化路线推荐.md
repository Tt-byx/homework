# Phase 4 完成：个性化路线推荐

## 提示词

> 现在开始做！！！（Phase 4 个性化路线推荐）

## 做了什么

### 1. 创建路线模板（5 条主题路线）

`scripts/import_routes.py` — 基于灵山胜境数据编写 5 条主题路线：

| 路线 | 标签 | 特色 |
|------|------|------|
| 历史文化深度游 | 历史/文化/佛教 | 灵山大佛 → 梵宫 → 五印坛城 → 九龙灌浴 → 阿育王柱 |
| 自然风光休闲游 | 自然/风光/休闲 | 大佛远观 → 菩提大道 → 太湖观景 → 拈花湾 → 花海 |
| 亲子互动体验游 | 亲子/家庭/互动 | 九龙灌浴 → 大佛登阶 → 梵宫 → 曼飞龙塔 → 拈花湾 |
| 摄影打卡精选路 | 摄影/打卡/拍照 | 大佛广场 → 菩提大道 → 梵宫 → 五印坛城 → 拈花湾夜景 |
| 美食禅意体验游 | 美食/禅意/体验 | 素食 → 梵宫 → 香月花街 → 禅茶 → 拈花湾晚餐 |

每条路线 8 个 chunk，共 40 个，已导入 ChromaDB（总计 223 chunks）。

### 2. 前端兴趣选择入口

`ChatView.vue` — 聊天区域顶部增加 5 个快捷标签按钮：
- 🏛️ 历史文化 → "我对历史文化感兴趣，请推荐一条游览路线"
- 🌿 自然风光 → "我喜欢自然风光，请推荐一条游览路线"
- 👨‍👩‍👧 亲子互动 → "我带小朋友来玩，推荐一条亲子路线"
- 📸 摄影打卡 → "我想拍照打卡，推荐一条摄影路线"
- 🍜 美食禅意 → "我想品尝美食和体验禅意文化，推荐一条路线"

标签仅在没有消息时显示，点击后发送预设文字，走 RAG 检索 + LLM 回答。

### 3. 优化推荐 Prompt

`rag_service.py` — system prompt 新增路线推荐格式要求：
- 先说明路线名称、适合人群、预计时长
- 按顺序列出每个站点（名称、时间、亮点）
- 最后给出温馨提示

### 4. 验证

- ✅ 5 条路线导入 ChromaDB 成功（223 chunks）
- ✅ `npm run build` — built in 895ms

## 为什么这样做

路线模板直接写入 ChromaDB，利用现有 RAG 检索链路，无需新增后端逻辑。前端快捷标签降低用户输入门槛。

## 产生的结果

游客可通过点击兴趣标签快速获取个性化路线推荐，AI 基于知识库中的路线模板生成结构化的游览建议。
