# 登录系统 + 历史对话侧边栏

## 提示词

> 我需要给这个引入登陆系统，就是引入用户的概念，然后右边增加一个历史对话的小窗口

## 做了什么

### 1. 后端认证系统

**AuthController.java** (新) — `/api/auth`:
- `POST /register` — 注册（username + password + nickname），自动登录返回 token
- `POST /login` — 登录，返回 token + 用户信息
- `GET /me` — 根据 Authorization header 返回当前用户
- 内存 Map 存储 token → userId（简单方案，不引入 JWT 库）

**ConversationController.java** (新) — `/api/conversations`:
- `GET /` — 当前用户的对话列表（按时间倒序）
- `GET /{id}/messages` — 某个对话的消息列表（验证归属权）

**ChatWebSocketHandler.java** — 改动：
- `afterConnectionEstablished` 解析 URL 参数 `?token=xxx` 获取 userId
- 创建 Conversation 时设置 `userId`

### 2. 前端用户系统

**stores/user.js** (新) — Pinia store：user/token 状态，login/register/logout/fetchMe

**api/auth.js** (新) — 认证 + 对话 API

**api/request.js** — 请求拦截器注入 `Authorization: Bearer <token>` header

**api/websocket.js** — WebSocket 连接时 URL 带 `?token=xxx`

### 3. 登录页面

**LoginView.vue** (新) — 简洁登录/注册表单，Element Plus 风格，登录成功跳转 /chat

### 4. 路由守卫

**router/index.js** — `/chat` 路由加 `meta: { requiresAuth: true }`，未登录自动跳转 `/login`

### 5. 历史对话侧边栏

**ChatView.vue** — 右侧新增 220px 宽的历史对话面板：
- 显示当前用户的所有对话（标题 + 时间）
- 点击对话加载该对话的消息
- "新对话"按钮清空聊天
- 对话完成后自动刷新列表
- 响应式：窄屏自动隐藏

**VisitorLayout.vue** — 顶栏显示用户名 + 退出按钮

## 编译验证

- ✅ Java `mvn compile` — BUILD SUCCESS
- ✅ 前端 `npm run build` — built in 1.34s

## 使用流程

1. 访问 `/login` 注册/登录
2. 登录后跳转 `/chat`，WebSocket 自动带 token
3. 发消息 → 对话自动关联用户 → 出现在右侧历史列表
4. 点击历史对话加载消息
5. 点"新对话"开始新会话
