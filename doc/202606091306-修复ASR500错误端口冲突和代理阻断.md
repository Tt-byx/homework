# 修复 ASR 500 错误：端口冲突 + 代理阻断 + 模型加载

## 提示词

> 现在的问题就是，输入语音输入，直接返回500

## 做了什么

### 1. 问题诊断

ASR 返回 500，有两个根因：

**根因 1：端口 8000 被 Windows Hyper-V 保留**
- 端口范围 7955-8054 被 Hyper-V 占用
- Python 后端无法绑定 8000 端口
- 改用端口 9000

**根因 2：系统代理阻断 modelscope 模型下载**
- 系统 socks5 代理 (127.0.0.1:7890) 阻止了 modelscope.cn 的连接
- FunASR 模型无法下载/验证，本地模式初始化失败
- 回退到远程模式（ws://localhost:10095），远程服务未运行 → 500
- 修复：加载模型时临时禁用代理

### 2. 具体改动

| 文件 | 改动 |
|------|------|
| `config.py` | python_port: 8000 → 9000 |
| `vite.config.js` | /py-api 代理目标: 8000 → 9000 |
| `application.yml` | python backend url: 8000 → 9000 |
| `asr_service.py` | init_local_model 时临时清除代理环境变量 |

### 3. 验证

```
FunASR local model loaded
POST /api/asr → 200 OK
```

## 为什么这样做

- Windows Hyper-V 随机动态保留端口范围，8000 恰好在保留范围内
- 系统 SOCKS5 代理会拦截所有出站连接，包括 modelscope.cn 的模型下载

## 产生的结果

- Python 后端在 9000 端口正常运行
- FunASR 本地模型成功加载（约 35 秒首次加载）
- ASR 端点正常返回识别结果
