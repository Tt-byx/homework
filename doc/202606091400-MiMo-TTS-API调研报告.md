# MiMo TTS API 接口调研报告

## 提示词
用户需要调研小米 MiMo TTS 模型 `mimo-v2.5-tts` 的 API 格式。提供的基本信息：
- Base URL: `https://token-plan-cn.xiaomimimo.com/v1`
- 声称是 OpenAI 兼容格式
- 需要确认端点、请求格式、响应格式

## 调研方法

1. 先通过 `/v1/models` 接口列出所有可用模型
2. 尝试标准 OpenAI TTS 端点 `/v1/audio/speech`（返回 404）
3. 发现 TTS 模型通过 `/v1/chat/completions` 端点调用，需要 assistant 消息
4. 逐一测试各种参数组合，确认完整接口规范
5. 同步测试了 ASR 模型

## 核心发现

### MiMo 平台可用模型列表

| 模型 ID | 类型 | 说明 |
|---|---|---|
| `mimo-v2-pro` | 文本 LLM | 大语言模型 |
| `mimo-v2-omni` | 多模态 | 多模态模型 |
| `mimo-v2-tts` | TTS | 语音合成 v2 |
| `mimo-v2.5` | 文本 LLM | 大语言模型 v2.5 |
| `mimo-v2.5-pro` | 文本 LLM | 大语言模型 v2.5 Pro |
| `mimo-v2.5-tts` | TTS | 语音合成 v2.5（推荐） |
| `mimo-v2.5-tts-voiceclone` | TTS | 声音克隆 TTS |
| `mimo-v2.5-tts-voicedesign` | TTS | 声音设计 TTS |
| `mimo-v2.5-asr` | ASR | 语音识别 |

### 重要发现：不是 `/v1/audio/speech`，而是 `/v1/chat/completions`

MiMo TTS **不使用** OpenAI 标准的 `/v1/audio/speech` 端点（该端点返回 404），而是复用 **`/v1/chat/completions`** 端点，通过特殊的 message 格式来传递待合成文本。

## TTS API 详细规范

### 端点
```
POST https://token-plan-cn.xiaomimimo.com/v1/chat/completions
```

### 请求头
```
Authorization: Bearer <API_KEY>
Content-Type: application/json
```

### 请求体格式
```json
{
  "model": "mimo-v2.5-tts",
  "messages": [
    {
      "role": "user",
      "content": ""
    },
    {
      "role": "assistant",
      "content": "这里是要合成语音的文本"
    }
  ]
}
```

**关键规则：**
- `messages` 数组中必须包含一个 `assistant` 角色的消息，其 `content` 就是要合成的文本
- `user` 角色的消息可以为空字符串
- 如果缺少 assistant 角色，会返回错误：`"messages must contain an assistant role for TTS model"`

### 响应格式
```json
{
  "id": "469760244a044bf987507361eb240060",
  "choices": [
    {
      "finish_reason": "stop",
      "index": 0,
      "message": {
        "content": "",
        "role": "assistant",
        "audio": {
          "id": "d5ff8090121e4fe7bd50f2eae7815073",
          "data": "<base64编码的WAV音频数据>",
          "expires_at": null,
          "transcript": null
        },
        "tool_calls": null,
        "final_text_preview": "..."
      }
    }
  ],
  "created": 1780987761,
  "model": "mimo-v2.5-tts",
  "object": "chat.completion",
  "usage": {
    "completion_tokens": 6,
    "prompt_tokens": 123,
    "total_tokens": 129,
    "completion_tokens_details": {
      "reasoning_tokens": 0
    },
    "prompt_tokens_details": {
      "cached_tokens": 121
    }
  }
}
```

### 音频数据格式

| 属性 | 值 |
|---|---|
| 格式 | WAV (RIFF) |
| 声道数 | 1（单声道） |
| 采样率 | 24000 Hz |
| 位深度 | 16 bit |
| Base64 编码 | `message.audio.data` 字段 |

### 性能测试数据

| 测试文本 | 音频时长 | 文件大小 | 耗时估算 |

| 故宫介绍段落（72字） | 2.56s | 123 KB | ~3s |

## ASR API 详细规范

### 端点
```
POST https://token-plan-cn.xiaomimimo.com/v1/chat/completions
```

### 请求体格式
```json
{
  "model": "mimo-v2.5-asr",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "input_audio",
          "input_audio": {
            "data": "data:audio/wav;base64,<base64编码的音频数据>",
            "format": "wav"
          }
        }
      ]
    }
  ]
}
```

**关键规则：**
- 音频数据必须使用 `data:audio/wav;base64,` 前缀的 Data URL 格式
- 直接传裸 base64 会报错：`"input_audio.data must be a valid data URL with mime type prefix"`
- 支持 wav 格式

### 响应格式
```json
{
  "id": "0e63c78d5d4e491093c2a8b316ab070b",
  "choices": [
    {
      "finish_reason": "stop",
      "index": 0,
      "message": {
        "content": "识别出的文字",
        "role": "assistant",
        "audio": null,
        "tool_calls": null,
        "audio_tokens": []
      }
    }
  ],
  "model": "mimo-v2.5-asr",
  "usage": {
    "completion_tokens": 9,
    "prompt_tokens": 28,
    "total_tokens": 37,
    "completion_tokens_details": {
      "reasoning_tokens": 0
    },
    "prompt_tokens_details": {
      "audio_tokens": 7,
      "cached_tokens": 4
    }
  }
}
```

## Python 调用示例

### TTS 调用示例
```python
import base64
from openai import OpenAI

client = OpenAI(
    api_key="your-api-key",
    base_url="https://token-plan-cn.xiaomimimo.com/v1"
)

# TTS 合成
response = client.chat.completions.create(
    model="mimo-v2.5-tts",
    messages=[
        {"role": "user", "content": ""},
        {"role": "assistant", "content": "你好，欢迎来到景区参观！"}
    ]
)

# 提取音频数据
audio_b64 = response.choices[0].message.audio.data
audio_bytes = base64.b64decode(audio_b64)

# 保存为 WAV 文件
with open("output.wav", "wb") as f:
    f.write(audio_bytes)
```

### ASR 调用示例
```python
import base64
from openai import OpenAI

client = OpenAI(
    api_key="your-api-key",
    base_url="https://token-plan-cn.xiaomimimo.com/v1"
)

# 读取音频文件并转为 data URL
with open("input.wav", "rb") as f:
    audio_b64 = base64.b64encode(f.read()).decode()

data_url = f"data:audio/wav;base64,{audio_b64}"

# ASR 识别
response = client.chat.completions.create(
    model="mimo-v2.5-asr",
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "input_audio",
                    "input_audio": {
                        "data": data_url,
                        "format": "wav"
                    }
                }
            ]
        }
    ]
)

recognized_text = response.choices[0].message.content
print(f"识别结果: {recognized_text}")
```

## 对项目的影响与建议

### 当前项目状态
- 项目配置了本地 ASR（FunASR）和本地 TTS（CosyVoice），需要本地部署模型服务
- MiMo 提供了云端 ASR 和 TTS 服务，可以**替代本地部署方案**，大幅简化部署复杂度

### 建议方案

**选项 A：使用 MiMo 云端 ASR/TTS（推荐）**
- 优势：无需本地部署 FunASR 和 CosyVoice，减少 Docker 镜像体积和 GPU 依赖
- ASR：使用 `mimo-v2.5-asr` 模型，通过 chat completions 端点调用
- TTS：使用 `mimo-v2.5-tts` 模型，通过 chat completions 端点调用
- 音频格式：WAV 24kHz 16bit 单声道

**选项 B：保持本地部署**
- ASR：本地 FunASR 服务
- TTS：本地 CosyVoice 服务
- 需要额外的 Docker 容器和 GPU 资源

**选项 C：混合方案**
- ASR 使用 MiMo 云端（延迟可接受）
- TTS 使用本地 CosyVoice（音色可定制，但部署复杂）

### 关键注意事项
1. MiMo TTS API 不是标准 OpenAI TTS 格式，需要通过 chat completions 调用
2. 音频输出固定为 WAV 24kHz 16bit 单声道
3. 目前未发现 voice 参数支持（传入后被忽略）
4. `mimo-v2.5-tts-voiceclone` 和 `mimo-v2.5-tts-voicedesign` 模型的接口格式需要进一步调研
