# 修复：端口确认、文档乱码、AI回复慢

## 提示词

> 1.java后端就要10086端口，python后端就好9000端口，相应的给我改回来
> 2.知识库中已上传的文档还是乱码的
> 3.ai回答问题很慢

## 做了什么

### 1. 端口确认 ✅

经检查，端口配置已经是正确的，无需修改：

| 组件 | 端口 |
|------|------|
| 前端 (Vite) | 5173 |
| Java 后端 | 10086 |
| Python 后端 | 9000 |

相关配置：
- `application.yml`: `server.port: 10086`, `python.backend.url: http://localhost:9000`
- `config.py`: `python_port: int = 9000`
- `vite.config.js`: `/api` → 10086, `/py-api` → 9000, `/ws` → ws://10086

### 2. 修复文档标题乱码 ✅

**根因**：Spring Boot 的 `MultipartFile.getOriginalFilename()` 返回 ISO-8859-1 编码的中文文件名，导致存储到数据库时变成乱码。

**修复**：
- `KnowledgeDocServiceImpl.java` 新增 `fixFilenameEncoding()` 方法：
  - 将文件名从 ISO-8859-1 重新解码为 UTF-8
  - 验证解码结果包含中文字符才采用
- `application.yml` 已配置 `server.servlet.encoding.charset: UTF-8, force: true`

**清理旧数据**：
- 删除了数据库 `knowledge_doc` 表中 3 条乱码记录（id=1,2,3）
- 清理了 ChromaDB 中对应的向量数据

### 3. 优化 AI 回复速度 ✅

**根因**：`ChatWebSocketHandler.handleTextChat()` 使用非流式的 `/api/chat` 接口，需要等 LLM 全部生成完 + 逐句 TTS 合成完，才能返回给前端。用户看到的是长时间空白等待。

**修复**：将文字对话改为流式 SSE 调用 Python 的 `/api/chat/stream`：
- 前端发送文字消息 → Java 用 `HttpURLConnection` 流式连接 Python 的 `/api/chat/stream`
- Python 端逐字生成文本（`stream=True`），每完成一句话就合成 TTS
- Java 逐行读取 SSE 事件，实时转发给前端（文字、音频、表情）
- 用户立刻看到文字逐字出现，不再需要等待全部生成完

**速度提升**：
| 对比项 | 修复前 | 修复后 |
|--------|--------|--------|
| 文字显示 | 等全部生成完（10-30秒） | 逐字实时显示（首字 < 3秒） |
| 音频播放 | 等全部 TTS 合成完 | 逐句播放（首句 < 5秒） |
| 用户体感 | 长时间无响应 | 实时流式反馈 |

## 使用方式

重新启动三端服务：

```bash
# 终端 1：Python 后端（端口 9000）
cd backend-python && uv run uvicorn main:app --port 9000

# 终端 2：Java 后端（端口 10086）
cd backend-java && mvn spring-boot:run

# 终端 3：前端（端口 5173）
cd frontend && npm run dev
```

**重要**：旧的乱码数据已清理，需要在管理后台重新上传知识库文档。
