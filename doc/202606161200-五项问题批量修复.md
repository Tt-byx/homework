# 五项问题批量修复

## 提示词

用户提出了5个问题：
1. AI的回答是md格式，给语音读出来不好，希望优化回答格式
2. 管理后台的前端UI太丑了，一看就是ai生成的，需要调用前端skill修改
3. 后台管理的数据大屏的数据和实际不太相符
4. 数据库中已上传的文件标题是乱码的，上传了3个文件（2个docx+1个xlsx），只有2个docx显示
5. 数字人的背景要修改，蓝紫色渐变色一看就是ai生成的

## 做了什么

### 1. AI回答格式优化（剥离Markdown）

**根因**: 系统提示词允许模型输出Markdown格式，TTS朗读时会读出`**`、`#`、`-`等符号。

**修复**:
- `backend-python/app/utils/text_utils.py` — 新增 `strip_markdown()` 函数，支持剥离标题、加粗、斜体、列表、链接、代码块、引用、表格等所有Markdown格式
- `backend-python/app/services/llm_service.py` — 更新 SYSTEM_PROMPT，明确禁止使用Markdown符号，要求口语化回答
- `backend-python/app/services/rag_service.py` — 更新 RAG 提示词，禁止Markdown，用"首先、其次"等自然语言替代编号列表
- `backend-python/app/routers/chat.py` — 在 `/chat` 和 `/chat/stream` 端点的TTS合成前调用 `strip_markdown()`，保留原始文字显示但语音使用纯文本

### 2. 管理后台UI重新设计

**根因**: 使用了蓝紫渐变、通用卡片样式、大量emoji标题等典型AI生成模板特征。

**修复**（调用 frontend-design skill）:
- `AdminLayout.vue` — 从深色侧边栏改为暖白底侧边栏，新增品牌标识区（景区智脑）、面包屑导航、收起菜单按钮，定义CSS设计令牌系统
- `DashboardView.vue` — 去掉所有emoji标题，统计卡片改为图标+文字布局（4种自然色调），图表使用绿色/暖棕色系，柱状图增加圆角
- `KnowledgeView.vue` — 去掉SVG装饰，统一使用CSS变量系统，表格样式与新设计语言一致
- `ConversationsView.vue` — 新增面板容器、记录计数标签、统一时间格式

设计语言：以 #5a8a6a（鼠尾草绿）为主色调，#f7f6f3 暖灰为页面背景，自然温暖不突兀。

### 3. 数据大屏数据修复

**根因**: WebSocket聊天路径（`ChatWebSocketHandler`）直接转发到Python，**没有将对话和消息保存到数据库**。只有REST API路径（`ChatServiceImpl`）才会持久化数据。因此数据大屏查询到的数据不完整。

**修复**:
- `ChatWebSocketHandler.java` — 注入 `ConversationMapper` 和 `ChatMessageMapper`
- `handleTextChat()` 方法中新增：
  1. 查询或创建 `Conversation` 记录（设置 status=1、createdAt、updatedAt）
  2. 保存用户消息（含 SentimentAnalyzer 情感分析）
  3. 保存AI回复（含情感分析）
- 每次WebSocket对话都会正确写入数据库，数据大屏数据将与实际交互一致

### 4. 文件上传标题乱码 + Excel缺失

**根因**: Spring Boot默认multipart编码为ISO-8859-1，中文文件名被错误编码导致乱码。

**修复**:
- `application.yml` — 新增 `spring.servlet.multipart.file-encoding: UTF-8`
- `WebConfig.java` — 新增 `CharacterEncodingFilter` Bean，强制UTF-8编码，解决multipart中文文件名乱码
- `vite.config.js` — 修正代理端口（/api → 10086, /py-api → 9000, /ws → 10086），之前指向错误端口

### 5. 数字人背景修改

**根因**: `.avatar-panel` 使用 `linear-gradient(135deg, #667eea, #764ba2)` 蓝紫渐变。

**修复**:
- `ChatView.vue` — 背景改为 `linear-gradient(180deg, #e8f4f8, #d4e9d7, #f0ebe3)` 模拟自然天空/草地/沙地的柔和过渡
- 表情栏背景从半透明白改为60%不透明白色，适配浅色背景
- 组标签文字从白色改为 `#666`

### 附带修复：端口配置还原

发现 `application.yml` 的端口被还原为 `port: 8080`、`python.backend.url: http://localhost:8000`，这正是上次"AI服务不可用"的根因。
- 修正为 `port: 10086`、`python.backend.url: http://localhost:9000`
- 同步修正 `vite.config.js` 代理端口

## 为什么这样做

- Markdown剥离在TTS前做而非提示词层面，是因为前端聊天气泡仍需保留格式显示（用户阅读体验），只在语音合成时剥离
- 管理后台设计采用自然色调系统（鼠尾草绿+暖灰），与景区主题呼应，避免AI模板感
- WebSocket数据持久化是关键bug修复，确保所有对话路径都写入数据库
- UTF-8编码过滤器是Spring Boot处理中文multipart的标准解决方案
