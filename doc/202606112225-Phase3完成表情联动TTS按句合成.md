## 做了什么

### 1. 数字人表情与 AI 回复情感联动 ✅

**全链路改动（Python → Java → 前端）：**

| 文件 | 改动 |
|------|------|
| `chat.py` (Python) | 新增 `detect_expression()` 关键词情感检测，返回表情名 |
| `chat.py` (schema) | ChatResponse 加 `expression` 字段 |
| `chat.py` (router) | `/chat` 返回 expression，`/chat/stream` 发送 expression SSE 事件 |
| `WebSocketMessage.java` | 加 `expression` 字段和工厂方法 |
| `ChatWebSocketHandler.java` | 转发 expression（非流式+流式两条路径） |
| `websocket.js` | 加 `expression` 事件处理 |
| `chat.js` (store) | 加 `currentExpression` ref 和 `onExpression` 回调 |
| `ChatView.vue` | watch `currentExpression` 自动切换 Live2D 表情 |

**情感映射规则：**
- 抱歉/无法/不知道 → Cry（哭泣）
- 恭喜/太好了/喜欢 → Smile（微笑）
- 注意/小心/禁止 → Angry（生气）
- ？/吗/什么 → Star（星星眼）
- 其他 → Normal（普通）

### 2. TTS 按句合成优化 ✅

**改动：**
- `/chat` 端点将完整回复按句号分句，逐句调用 TTS
- 返回逗号分隔的多个 base64 音频
- Java handler 拆分并逐个发送 `audio_chunk` 事件
- 效果：首句 1-2 秒即可播放，无需等全部合成完

### 3. 口型同步效果调试

需要在浏览器中实际测试，代码逻辑已就绪：
- 文字口型：`viseme-mapper.ts` 汉字韵母→嘴形映射
- 音频口型：`AudioPlayer.js` AnalyserNode → RMS → 口型值
- 优先级：音频播放时停文字口型，音频播完恢复
