# 20260619194500-Phase7消费分析与反馈系统实现.md

## 本次做了什么

实现了 Phase 7 的全部三个子任务：xlsx 结构化数据导入、消费分析看板、反馈按钮系统。

## 为什么这样做

对照竞赛 16 条需求，xlsx 消费分析（需求 #8）、可视化报表（需求 #5）、快捷按钮（需求 #10）是功能完整度的硬性扣分项。xlsx 文件包含 140,448 行 × 17 列的游客行为数据，是消费分析的核心数据来源。

## 改动详情

### 7.1 xlsx 结构化数据导入 + 消费分析 API

**数据库：**
- 新建 `tourism_data` 表（17 个字段，含索引）
- 新建 `message_feedback` 表（反馈系统）

**Java 后端：**
| 文件 | 操作 | 说明 |
|------|------|------|
| `entity/TourismData.java` | 新建 | 消费数据实体 |
| `mapper/TourismDataMapper.java` | 新建 | MyBatis-Plus Mapper |
| `entity/MessageFeedback.java` | 新建 | 反馈数据实体 |
| `mapper/MessageFeedbackMapper.java` | 新建 | 反馈 Mapper |
| `controller/AnalyticsController.java` | 新建 | 消费分析 API（import + 6 个查询） |
| `controller/FeedbackController.java` | 新建 | 反馈 API（提交 + 统计） |
| `service/AnalyticsService.java` | 新建 | 服务接口 |
| `service/impl/AnalyticsServiceImpl.java` | 新建 | Apache POI 解析 xlsx + SQL 聚合查询 |
| `pom.xml` | 修改 | 添加 Apache POI 依赖 |

**API 端点：**
| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/analytics/import` | POST | 解析 xlsx 导入数据库 |
| `/api/analytics/consumption` | GET | 按月消费趋势 |
| `/api/analytics/visitor-profile` | GET | 游客画像（年龄/性别/团队） |
| `/api/analytics/satisfaction` | GET | 满意度分布 |
| `/api/analytics/peak-periods` | GET | 按月客流 |
| `/api/analytics/attractions` | GET | 热门景点 Top 10 |
| `/api/feedback` | POST | 提交点赞/踩 |
| `/api/feedback/stats` | GET | 反馈统计 |
