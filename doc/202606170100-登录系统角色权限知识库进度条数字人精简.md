# 登录系统 + 角色权限 + 知识库进度条 + 数字人精简

## 提示词

> 我需要给这个引入登陆系统，引入用户的概念，右边增加历史对话小窗口
> 登陆界面做好看点，调用前端skill
> 做角色划分，只有管理员才能看后台
> 知识库改为上传和处理两步骤，都给出进度条
> 数字人不需要交互动作，只要微微抖动

## 做了什么

### 1. 登录/注册系统
- AuthController: 注册、登录、获取当前用户 (UUID token 认证)
- User store + auth API + 请求拦截器注入 token
- LoginView: 左右分栏水墨山水风格（桌面端），小屏自动切换居中卡片

### 2. 角色权限
- 路由守卫: /chat 需登录，/admin 需 admin 角色
- VisitorLayout: 按角色显示"管理后台"按钮
- 预置管理员账号: admin / admin123

### 3. 历史对话侧边栏
- ConversationController: 用户对话列表 + 消息查询
- ChatView 右侧 220px 面板，点击加载历史对话，新建对话按钮
- WebSocket 连接时传 token，创建对话关联 userId

### 4. 知识库两步式处理
- 上传和处理分离，进度条实时更新（parsing→chunking→embedding→storing）
- Python SSE 流式返回进度
- text_chunker 限制最大 chunk 数，embedding 限流重试

### 5. 数字人精简
- 去除所有交互事件和控制按钮
- 保留微微抖动动画（正弦波位移）
- 去除 console.log
