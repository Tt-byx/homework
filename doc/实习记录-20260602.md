二、存在的问题

1. MySQL JDBC 编码问题
   在 application.yml 的数据库连接 URL 中，最初写了 characterEncoding=utf8mb4，启动时报错 UnsupportedEncodingException: utf8mb4。排查后发现 utf8mb4 是 MySQL 特有的字符集名称，Java JDBC 只认 UTF-8，改为 characterEncoding=UTF-8 后解决。

2. Java Bean Validation 依赖缺失
   在 ChatRequest DTO 中使用了 @NotBlank 注解，编译时报错找不到 jakarta.validation 包。原因是 Spring Boot 3.x 将 validation 从 web starter 中拆分出来了，需要单独引入 spring-boot-starter-validation 依赖。
