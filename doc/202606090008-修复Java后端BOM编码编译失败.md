# 修复 Java 后端 BOM 编码导致编译失败

## 提示词

> java后端启动报错，然后我更新了CLAUDE.md文件，要求你有修改代码的行为时候，先编译通过，确保所有功能正常才返回给用户

## 做了什么

### 1. 问题诊断

Java 后端 `mvn compile` 报错，3 个文件有 UTF-8 BOM（`﻿`）：
- `WebSocketConfig.java`
- `ChatWebSocketHandler.java`
- `WebSocketMessage.java`

BOM 是 Windows 上常见的编码问题，Java 编译器不认识文件开头的 BOM 字节。

### 2. 修复

用 Python 去除 3 个文件的 BOM：
```python
with open(f, 'rb') as fh:
    data = fh.read()
if data[:3] == b'\xef\xbb\xbf':
    with open(f, 'wb') as fh:
        fh.write(data[3:])
```

### 3. 验证

三端全部编译/构建通过：
- ✅ Java `mvn compile` — BUILD SUCCESS（25 个源文件）
- ✅ Python `uvicorn main:app` — 启动成功，ChromaDB 初始化成功
- ✅ 前端 `npm run build` — built in 905ms

## 为什么这样做

新拉取的 GitHub 代码在 Windows 上被 Git 自动加了 BOM，导致 Java 编译器报"非法字符"错误。

## 产生的结果

Java 后端编译通过，可以正常启动。
