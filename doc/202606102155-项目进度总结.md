## 一、工作进展

### Phase 0：项目骨架搭建 ✅ 已完成
- Vue3 前端（游客端 + 管理后台）
- Spring Boot 后端（Java 17，端口 10086）
- FastAPI 后端（Python 3.13，端口 9000）
- MySQL 数据库（5 张表：用户、景点、会话、消息、知识文档）
- 文字对话全链路跑通

### Phase 1：RAG 知识库构建 ✅ 已完成
- 文档解析（docx/xlsx）→ 文本切片（300-500字，50字重叠）
- Embedding（SiliconFlow BAAI/bge-m3）→ ChromaDB 存储（183 个 chunk）
- RAG 检索链路（Top-K=5 → 防幻觉 Prompt → MiMo 生成）
- 测试集 20 个问题，准确率 90%（18/20）
- 管理后台知识库管理页面（上传/列表/删除/重处理/状态轮询）

### Phase 2：语音交互链路 ✅ 基本完成
- **ASR 语音识别**：MiMo mimo-v2.5-asr API
- **TTS 语音合成**：MiMo mimo-v2.5-tts API
- **语音输入**：录音（AudioContext WAV 16kHz）→ ASR → 文字填入输入框
- **语音输出**：LLM 回答 → TTS → 音频通过 WebSocket 发送给前端播放
- **流式对话**：WebSocket 双向通信（前端 ↔ Java ↔ Python）

### Phase 3：数字人形象与口型同步 🔄 部分完成（新增）
- **Live2D 数字人**：集成 Cubism SDK，white 角色模型（64MB 纹理）
- **口型同步**：文字口型（viseme-mapper 汉字→口型映射）+ 音频口型（AnalyserNode RMS）
- **表情系统**：12 种表情/姿势（普通、微笑、星星眼、哭泣、生气等）
- **交互**：点击/长按数字人，眼球跟随鼠标
- **设置面板**：眼球/头部/身体灵敏度、呼吸幅度
- **ChatView 改版**：左侧数字人 + 右侧聊天，响应式布局

## 二、存在的问题

1. **数字人与语音的口型优先级**：文字口型和音频口型的切换逻辑已实现，但实际效果需要在浏览器中测试
2. **TTS 延迟**：MiMo TTS API 每次调用约 1-2 秒，长文本回答可能需要 3-5 秒才能播放完整语音
3. **WebSocket 非流式**：Java handler 用 RestTemplate 同步调用 Python，文字回答是一次性返回而非流式
4. **RAG prompt 是英文**：`rag_service.py` 的 `build_prompt` 用了英文模板，可优化为中文
5. **管理后台数据大屏**：Phase 5 未实现，DashboardView 仍是占位
6. **用户认证**：无登录/注册功能，游客和管理员未区分
