**时间：** 2026-06-02 17:30

## 提示词

用户请求：开始 Phase 0，搭建 Vue3 + Spring Boot + FastAPI 三端项目骨架，跑通"文字输入 → 大模型回答 → 文字显示"全链路。

---

## 做了哪些事情

### 1. 创建数据库和表
- 建库 `scenic_ai`（utf8mb4）
- 建 5 张表：`user`、`scenic_spot`、`conversation`、`chat_message`、`knowledge_doc`
- DDL 同时保存到 `backend-java/src/main/resources/schema.sql`

### 2. 初始化 Python 后端（FastAPI + MiMo API）
- 使用 `uv init` + `uv add` 安装依赖（fastapi, uvicorn, openai, pydantic-settings, httpx）
- 创建项目结构：main.py、app/core/config.py、app/schemas/chat.py、app/services/llm_service.py、app/routers/chat.py
- 配置 MiMo API（OpenAI 兼容格式）：`https://token-plan-cn.xiaomimimo.com/v1`，模型 `mimo-v2.5-pro`
- .env 文件存放 API Key（已在 .gitignore 中）

### 3. 初始化 Spring Boot 后端
- 手动创建 Maven 项目：Spring Boot 3.3.5 + MyBatis-Plus 3.5.7 + MySQL 8.0 + WebSocket
- 创建完整 Java 类：启动类、配置类（CORS/MyBatis-Plus/WebSocket）、Controller、Service、Entity（5个）、Mapper（5个）、DTO（3个）
- 聊天接口 `POST /api/chat`：接收前端消息 → 创建会话 → 保存消息 → 转发 Python 后端 → 保存 AI 回复 → 返回结果
- 修复了两个问题：
  - 缺少 `spring-boot-starter-validation` 依赖（jakarta.validation 报错）
  - MySQL 连接 URL 中 `characterEncoding=utf8mb4` 应为 `UTF-8`

### 4. 初始化 Vue 3 前端
- `npm create vite@latest frontend -- --template vue` + 安装 vue-router、pinia、axios、element-plus
- 实现双布局：游客端（头部 + 聊天页面）和管理后台（侧栏 + 内容区）
- 核心聊天组件：ChatMessage（消息气泡）、MessageList（消息列表）、ChatInput（输入框）
- Pinia store 管理聊天状态
- Vite 配置 proxy 转发 `/api` 到 Java 后端 8080

### 5. 全链路联调验证
- 三个服务全部启动：Python(8000)、Java(8080)、Vue(5173)
- 通过前端 proxy 测试完整链路：
  - 前端(5173) → Vite proxy → Java(8080) → Python(8000) → MiMo API → 原路返回
- 数据库验证：conversation 表和 chat_message 表记录正确

- 提交信息：`feat: Phase 0 初始化三端项目骨架，跑通文字对话全链路`

## 修复的问题

1. **`jakarta.validation` 缺失**：pom.xml 中缺少 `spring-boot-starter-validation` 依赖，添加后解决
2. **`UnsupportedEncodingException: utf8mb4`**：MySQL JDBC URL 中 `characterEncoding=utf8mb4` 不是合法 Java charset，改为 `UTF-8`
3. **Pydantic 请求体解析**：`session_id` 字段名与 Java 端 `sessionId` 存在不一致，后续需注意两端字段命名对齐
