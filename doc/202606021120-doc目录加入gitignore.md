# 将 doc/ 目录加入 .gitignore

**时间：** 2026-06-02 11:20

---

## 提示词

用户请求：该项目下的 doc 文件无需上传到 GitHub，修改 .gitignore，然后重新上传。

---

## 做了哪些事情

### 1. 检查当前状态
- 当前在 `docs/init-setup` 分支
- `doc/` 目录下有一个已跟踪文件：`doc/202606021027-git仓库初始化与远程连接.md`
- 还有一个未跟踪文件：`doc/202606021115-移除Playwright-MCP配置.md`

### 2. 切换到 main 分支
```bash
git checkout main
```

### 3. 修改 .gitignore
在 `.gitignore` 中新增 `doc/` 忽略规则：
```gitignore
# ========================
# 项目文档（本地记录，不上传）
# ========================
doc/
```

### 4. 从 Git 中移除已跟踪的 doc 文件
```bash
git rm --cached doc/ -r
```
- 使用 `--cached` 仅从 Git 索引中移除，不删除本地文件
- 移除了 `doc/202606021027-git仓库初始化与远程连接.md`

### 5. 提交并推送
```bash
git commit -m "chore: 将 doc/ 目录加入 .gitignore，不上传到远程"
```
- 远程有新提交，先 `git pull --rebase` 合并
- 推送成功到 `origin/main`

---

## 产生结果

- `doc/` 目录已加入 `.gitignore`，后续新增的 doc 文件不会被 Git 跟踪
- 远程仓库中已删除 `doc/202606021027-git仓库初始化与远程连接.md`
- 本地 `doc/` 目录及所有文件仍保留，不受影响
- 推送成功：`68bb0d8..c38a2f2 main -> main`
