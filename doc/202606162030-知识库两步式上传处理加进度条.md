# 知识库改为两步式上传处理 + 进度条

## 提示词

> 知识库中没有显示之前处理的文档，我觉得应该改为上传和处理两步骤，并且都给出进度条告诉用户处理到哪一步了。

## 做了什么

### 1. 数据库迁移 ✅

`knowledge_doc` 表新增两列：
- `process_stage` VARCHAR(50) — 处理阶段（parsing/chunking/embedding/storing/completed）
- `process_progress` INT — 处理进度 0-100

### 2. Java 后端重构 ✅

**KnowledgeDoc.java** — 新增 `processStage` 和 `processProgress` 字段

**KnowledgeDocServiceImpl.java** — 重写为两步式：
- `upload()` — 只保存文件到磁盘 + 创建 DB 记录（vectorStatus=0 待处理），**不触发处理**
- `process(Long id)` — @Async 异步调用 Python SSE 流式接口，逐阶段更新 DB 进度

**KnowledgeDocController.java** — 新增 `POST /api/knowledge/process/{id}` 端点

### 3. Python 后端 SSE 流式进度 ✅

**knowledge.py** — `/api/knowledge/process` 改为 StreamingResponse（SSE）：
- `event: progress` — 每个阶段的进度更新（stage + progress + message）
- `event: done` — 处理完成（含 chunk_count）
- `event: error` — 处理失败

向量化阶段按 batch（32个）逐批报告进度，精确到百分比。

### 4. 前端重构 ✅

**knowledge.js** — 新增 `processKnowledge(id)` API，`uploadKnowledge` 支持 onUploadProgress 回调

**KnowledgeView.vue** — 完全重写：
- **Step 1 上传区域**：拖拽上传 + 标题输入 + 上传按钮（带实时进度百分比）
- **Step 2 文档列表**：
  - "待处理"状态 → 显示"开始处理"按钮
  - "处理中"状态 → 显示进度条 + 当前阶段文字（如"向量化 65%"）
  - "已完成"状态 → 绿色勾 + chunk 数量
  - "失败"状态 → 红色叉 + 重试按钮
- 1秒轮询（处理中时），完成后自动停止

## 编译验证

- ✅ Java `mvn clean compile` — BUILD SUCCESS
- ✅ 前端 `npm run build` — built in 1.35s

## 使用流程

1. 用户选择文件 + 填写标题 → 点击"上传文件"（带上传进度条）
2. 上传完成后文档出现在列表，状态为"待处理"
3. 用户点击"开始处理" → 进度条从 0% 到 100%（解析→切分→向量化→存储）
4. 处理完成 → 显示"已完成 (223 片段)"
