# 修复 ASR 500：浏览器录音格式转换问题

## 提示词

> 语音识别还是失败，返回500状态码

## 做了什么

### 1. 问题诊断

浏览器 MediaRecorder 录音输出 **webm/opus** 格式，但 FunASR 只支持 **wav/pcm**。后端收到 webm 数据直接写入 .wav 文件交给模型，格式不识别 → 500。

### 2. 修复方案

**重写 VoiceRecorder.js**：不再用 MediaRecorder（输出 webm），改用 AudioContext + ScriptProcessorNode 录制原始 PCM 数据，在前端编码为 WAV 格式。

技术细节：
- AudioContext 以 16kHz 采样率创建（与 FunASR 匹配）
- ScriptProcessorNode 每 4096 采样收集一次 Float32 PCM 数据
- 停止录音时合并所有 PCM 块，Float32 → Int16，加上 WAV 文件头
- 输出标准 16kHz 16bit 单声道 WAV，可直接被 FunASR 识别

### 3. 同时保留后端转换

`chat.py` 的 ASR 端点保留了 `_convert_to_wav` 函数（用 torchaudio），作为其他格式的 fallback。

### 4. 验证

- ✅ `npm run build` — built in 968ms

## 为什么这样做

安装 ffmpeg 可以解决格式转换，但增加了部署依赖。在前端直接录 WAV 是零依赖方案。

## 产生的结果

语音输入全链路：录音（WAV 16kHz）→ POST 到 /py-api/asr → FunASR 识别 → 文字填入输入框。
